## 環境

* PC
    - Lenovo IdeaPad
* Arch Linux isoファイル
    - https://mirrors.cat.net/archlinux/iso/2026.04.01/archlinux-x86_64.iso

## インストールメディアの作成
この段階でかなり苦戦しまして、、、

最初、iso焼くときはいつもお世話になっている`balena etcher`を使って焼いたんですが起動しない、、、というか`UEFI`に認識されない(Boot orderを変更できない)。

`balena etcher`がよくないのかと思い、次に試したのが`rasberry pi imager`でした。が、それも同じ結果に。。。


何故だろう？考えたときに、一つ心当たりがありました。それは、**PCがUEFI環境だったことです**。今まで`BIOS`のPCしか使ってこなかったので盲点でした。

で、少し調べると、**UEFIのPCでディスクを認識させるためには、パーティションスタイルが<u>GPT</u>でなければいけない**ということが分かりました。
`rasberry pi imager`でisoを書き込んだあとのUSBを確認すると、`MBR`となっていました。試しに、windowsでGPT作成したあとで`rasberry pi imager`を使ってみましたが、`MBR`に上書きされてしまっていました。`Balena Etcher`も同様に`MBR`に上書きしてしまいます。

### なら、rufusを使おう！
`rufus`を使えば、パーティションスタイルを選べる！！！`GPT`を選択して、書き込み、UEFIを開くと見事にUSBが認識されていました！


## インストールメディア起動
まさかのインストールメディア作成が一番の山場だったんです、、、インストールは結構スムーズにできました！


とりあえずこの公式wikiを読んでそのとおりに進めていけば間違いないと思います。
https://wiki.archlinux.jp/index.php/%E3%82%A4%E3%83%B3%E3%82%B9%E3%83%88%E3%83%BC%E3%83%AB%E3%82%AC%E3%82%A4%E3%83%89

ちなみに、ネットワークはandroidでテザリングしました。有線も無線も面倒なときはテザリングが**だいたい**解決してくれます。


また、Arch Linuxのインストールで重要なことを色々してくれる(?)このコマンド↓
```
$ pacstrap -K /mnt base linux linux-firmware
```
これを実行する前に、**`UEFI`の人は、`esp`を作っておいたほうがいいかも**。(AI情報のため真偽は不明だけど)

私は、上記コマンドを実行したあとに`esp`を作成したためなのか、インストール後の3回目の起動くらいで/bootが空っぽになっていました。(謎)

`UEFI`にも認識されなかったのであせりました。手元にあったArchインストールメディアで`esp`と`/boot`(grub)を作り直したらなんとか治りました。

### テキストエディタ使えなくない?

公式wikiのインストール手順では`/etc/locale.conf`や`/etc/hostname`を編集しないといけないときがあるんですが、`vim`も`nano`も使えない (インストールされていない)。

`sed`や`awk`も使えない私は最初、どうやって編集すればいいのか分からず迷いましたが、`echo`の出力をリダイレクトする力技で何とかしました。

例えば、`locale-gen`を実行するために`/etc/locale.conf`の`LANG=en_US.UTF-8`という行をコメントアウトする作業は、こんな感じです↓
```
$ echo  'LANG=en_US.UTF-8' > /etc/locale.conf ## >> でもOK
```
コメントアウトと言っていますが、`>`を使用しているためファイル内容を上書きしています。

`>>`を使用すれば元のコメントされている設定を破壊せずにいけますね。

また、私の場合は日本語も使いたいので`LANG=ja_JP.UTF-8`も追加してこうしました↓
```
$ echo -e "LANG=en_US.UTF-8\nLANG=ja_JP.UTF-8" > /etc/locale.conf
```
