HUGOで画像が読み込めない時に確認すべきことをまとめました。私自身、画像が読めないトラブルが発生しがちなので。

## 1. 画像へのPATHはあっているか
言わずもがなですが、よくあるのでちゃんと確認しましょう。ファイル名は一語一句大文字小文字一致している必要があります。

例えば、/static/img/に画像を置いているなら、こうかきます。
```
![alternate text](/img/img.png)
```
自分の環境に合わせてPATHを書きましょう。
## 2. 画像ファイルは存在しているか 
これ以外とあります。一度`hugo server`をして画像PATHにアクセスすることで存在を確認できるのでやってみてください。
## 3. マークダウンの記法は正しいか
このように書いていませんか？
```
![alternate text]("/path/to/image")
```
正しくはこうです。
```
![alternate text](/path/to/image)
```

**PATHはダブルクオーテーションで囲ってはいけません**。シングルクォーテーションもだめです。

マークダウンの詳しい記法はこちらのサイトが参考になります。
https://www.markdownguide.org/basic-syntax/

## 4. インターネット上の画像の場合
* URLを表示したらその画像だけが表示されるか
* 画像ファイルが消えていないか

確認しておきましょう。
