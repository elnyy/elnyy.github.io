(widows) FirefoxでArkenfoxのuser.jsを設定

April 27, 2026
Windows, Firefox

arkenfox user.jsとは

firefoxの設定をできる限りプライバシー重視に設定したい人向けのuser.jsである。

ソースコードは以下のURLから確認できる。

https://github.com/arkenfox/user.js/

導入方法

普通にuser.jsを設定する方法と同じように可能。

  1. まず、URLバーからabout:profilesにアクセスする。

  2. 現在使用中のプロファイルを見つけたら、Local Directoryではなく、Root Directoryの方のフォルダを開く。

  3. 開いたフォルダ内で、user.jsという名前でファイルを作る。

  4. 以下のarkenfoxのuser.jsをコピーし、作成したuser.jsに貼り付ける。

https://raw.githubusercontent.com/arkenfox/user.js/refs/heads/master/user.js

  1. firefoxの全てのタブを終了し、再度起動することで設定が反映されるはず。

設定が反映されない時は

  • プロファイルは間違っていないか
    • about:profileを再度確認してみる
  • ファイル名がuser.jsとなっているか
    • 一語一句違ってはいけない
  • 再起動できていない
    • Windows自体を再起動してみる

初期の設定だと、自分は使っていないアカウントメニューだとかが表示されていて鬱陶しいんですよ。 これをすれば見た目がスッキリするのでいいですね。

あと、極端にウェブサイトが機能しなくなるってこともありませんし。

Arch Linux をUEFI環境にインストールしたときのメモ

April 23, 2026
Linux

先日、初めてArch Linuxに手を出しました。 Archはインストールが難しそうという偏見からずっと避けてきました。 ですが、最近普段から使っていたdevuanが色々あって使えなくなってしまったので、心機一転、重い腰を上げてArchデビューしてみようと思った次第です。

今回は、Arch linuxインストール中に出会った初見殺しや注意点をメモとして書いていきます。

環境

  • PC
    • Lenovo IdeaPad
  • Arch Linux isoファイル
    • https://mirrors.cat.net/archlinux/iso/2026.04.01/archlinux-x86_64.iso

インストールメディアの作成

この段階でかなり苦戦しまして、、、

最初、iso焼くときはいつもお世話になっているbalena etcherを使って焼いたんですが起動しない、、、というかUEFIに認識されない(Boot orderを変更できない)。

balena etcherがよくないのかと思い、次に試したのがrasberry pi imagerでした。が、それも同じ結果に。。。

何故だろう？考えたときに、一つ心当たりがありました。それは、PCがUEFI環境だったことです。今までBIOSのPCしか使ってこなかったので盲点でした。

で、少し調べると、UEFIのPCでディスクを認識させるためには、パーティションスタイルがGPTでなければいけないということが分かりました。 rasberry pi imagerでisoを書き込んだあとのUSBを確認すると、MBRとなっていました。試しに、windowsでGPT作成したあとでrasberry pi imagerを使ってみましたが、MBRに上書きされてしまっていました。Balena Etcherも同様にMBRに上書きしてしまいます。

なら、rufusを使おう！

rufusを使えば、パーティションスタイルを選べる！！！GPTを選択して、書き込み、UEFIを開くと見事にUSBが認識されていました！

インストールメディア起動

まさかのインストールメディア作成が一番の山場だったんです、、、インストールは結構スムーズにできました！

とりあえずこの公式wikiを読んでそのとおりに進めていけば間違いないと思います。 https://wiki.archlinux.jp/index.php/%E3%82%A4%E3%83%B3%E3%82%B9%E3%83%88%E3%83%BC%E3%83%AB%E3%82%AC%E3%82%A4%E3%83%89

ちなみに、ネットワークはandroidでテザリングしました。有線も無線も面倒なときはテザリングがだいたい解決してくれます。

また、Arch Linuxのインストールで重要なことを色々してくれる(?)このコマンド↓

$ pacstrap -K /mnt base linux linux-firmware

これを実行する前に、UEFIの人は、espを作っておいたほうがいいかも。(AI情報のため真偽は不明だけど)

私は、上記コマンドを実行したあとにespを作成したためなのか、インストール後の3回目の起動くらいで/bootが空っぽになっていました。(謎)

UEFIにも認識されなかったのであせりました。手元にあったArchインストールメディアでespと/boot(grub)を作り直したらなんとか治りました。

テキストエディタ使えなくない?

公式wikiのインストール手順では/etc/locale.confや/etc/hostnameを編集しないといけないときがあるんですが、vimもnanoも使えない (インストールされていない)。

sedやawkも使えない私は最初、どうやって編集すればいいのか分からず迷いましたが、echoの出力をリダイレクトする力技で何とかしました。

例えば、locale-genを実行するために/etc/locale.confのLANG=en_US.UTF-8という行をコメントアウトする作業は、こんな感じです↓

$ echo  'LANG=en_US.UTF-8' > /etc/locale.conf ## >> でもOK

コメントアウトと言っていますが、>を使用しているためファイル内容を上書きしています。

>>を使用すれば元のコメントされている設定を破壊せずにいけますね。

また、私の場合は日本語も使いたいのでLANG=ja_JP.UTF-8も追加してこうしました↓

$ echo -e "LANG=en_US.UTF-8\nLANG=ja_JP.UTF-8" > /etc/locale.conf

お疲れ様

タイトルで「インストールしたときのメモ」とか言いつつ、記事の大半がインストールメディア作成の内容なのは許して。 インストール自体はwikiを見れば特に問題なくできるとは思います。とはいえ、インストール完了後、rebootして最初に起動するかどうかを確認する際の緊張感は半端ないですね。でもちゃんと起動してGrub画面が見れた時はめちゃくちゃ嬉しかったです。

(Linux) exitコマンドとは？使い方をわかりやすく解説

April 18, 2026

Linuxを使っていると、ターミナルやSSH接続の終了時に「exit」というコマンドによくお世話になります。このexitは非常にシンプルですが、システム操作において重要な役割を持っています。しかし、初心者のうちはいつ使うのか分かりにくいコマンドのひとつかもしれません。

exitとは

exitは、その名前の通り「終了する」ためのコマンドです。Linuxのシェル（ターミナルの操作環境）やSSH接続、スクリプトの実行など、さまざまな場面で現在のセッションを終了するために使われます。

つまり、作業している環境から抜けるときに使う基本的なコマンドです。

基本的な使い方

引数いらずシンプルに実行です。 コマンド例

$ exit

このコマンド実行すると、現在開いているシェルが終了します。ターミナルを直接開いている場合はウィンドウが閉じるか、元の状態に戻ります。

SSH接続でのexitの使い方

リモートサーバーにSSH接続している場合、exitは特に重要です。

例えば次のような状況です。

$ ssh user@server

このようにサーバーへログインしたあと、作業が終わったら

$ exit

と入力することで、リモートサーバーとの接続が切断され、ローカル環境に戻ることができます。

これはログアウト操作とほぼ同じ意味になります。

複数のシェルを開いている場合

ターミナルの中でさらにシェルを起動している場合、exitはその一つ一つの階層を抜けるために使われます。

例えば

$ bash

で新しいシェルに入った場合、exitを実行すると一段階前のシェルに戻ります。さらにもう一度exitを実行すると、最初のターミナルに戻るというように、階層的に終了していきます。

exitの戻り値

exitでは終了ステータスを引数として渡すことができます。

$ exit 0

このように数字を指定することで、終了時の状態をシステムに伝えることができます。一般的に0は正常終了、それ以外の数字はエラーを意味します。

スクリプトの中でもよく使われる重要な概念です。

シェルスクリプトでのexit

bashスクリプトの中でもexitは頻繁に使われます。

例えば

ある条件で処理を終了したい場合や、エラーが発生した時点で処理を止めたい場合などに使います。

$ exit 1

のように書くことで、その時点でスクリプト全体の実行を終了できます。

注意点

exitは非常にシンプルですが、実行するとそのセッションが終了するため注意が必要です。

特にSSH接続中に作業途中でexitを実行してしまうと、その時点で接続が切れてしまい、未保存の作業が失われる可能性があります。

また、複数の作業を同時に行っている場合は、どのシェルを終了しているのかを意識することも重要です。

まとめ

exitはLinuxの中でも非常に基本的なコマンドでありながら、システム操作の終了という重要な役割を持っています。

  • シェルの終了
  • SSH接続のログアウト
  • スクリプトの終了制御

このようにさまざまな場面で使われるため、Linuxを扱う上では必ず理解しておきたいコマンドのひとつです。

特に初心者のうちは「とりあえず抜けたいときに使うコマンド」として覚えておくと便利ですが、慣れてきたら終了ステータスなども使いこなせるようになるといいです。

(Linux) catコマンドとは？使い方をわかりやすく解説

April 18, 2026

Linuxを使い始めると、最初の方で必ず出てくるコマンドのひとつがcatコマンドです。とてもシンプルなコマンドですが、テキストファイルの内容を見たいときなど、日常的な作業で頻繁に使われます。初心者でも覚えやすい一方、実務でも役に立つ重要なコマンドです。

catコマンドとは

catは「concatenate（連結する）」の略です。本来は複数のファイルをつなげるためのコマンドですが、現在では主にファイルの中身を表示する用途で使われることが多いです。

Linuxの基本操作の中でも使用頻度が高く、最初に覚えておくべきコマンドのひとつです。

catコマンドでファイルの中身を表示する

最も基本的な使い方は、ファイルの内容を表示することです。

  • コマンド例 $ cat ファイル名

例えば、sample.txtというファイルの中身を見たい場合は次のように入力します。

$ cat sample.txt

実行すると、ターミナル上にファイルの中身がそのまま表示されます。非常にシンプルで直感的な使い方ですね。

複数ファイルの内容をまとめて表示する

複数のファイルを同時に扱うこともできます。

  • コマンド例
$ cat sample1.txt sample2.txt

この場合、sample1.txtの内容が表示されたあとにsample2.txtの内容が続けて表示されます。ファイルをまとめて確認したいときに便利です。

新しいファイルを作成する

ファイルを作成することもできます。

  • コマンド例
$ cat > newsample.txt

このコマンドを実行すると入力待ち状態になり、そこに文字を入力していきます。入力が終わったらCtrl + Dを押すことで保存されます。

ちょっとしたメモ作成などに使える方法です。

既存ファイルに追記する

すでに存在するファイルに内容を追加したい場合は次のようにします。

  • コマンド例
$ cat >> sample.txt

この状態で入力した内容は、sample.txtの末尾に追加されます。ログの追記や簡単な記録追加などに使えます。

ファイルの結合

catコマンドの本来の用途であるファイル結合も重要です。

  • コマンド例
$ cat sample1.txt sample2.txt > merged.txt

このコマンドを実行すると、sample1.txtとfile2.txtの内容が結合され、新しくmerged.txtとして保存されます。

複数のテキストファイルをひとつにまとめたいときに非常に便利です。

便利なオプション

痒い所に手が届くオプションがありますので使いこなせるようになりましょう

  • 行番号を表示する
$ cat -n sample.txt

プログラムの確認やログ解析のときに役立ちます。

  • 空行をまとめる
$ cat -s sample.txt

連続した空行を1行にまとめて表示できます。

  • 制御文字も表示する
$ cat -A sample.txt

タブや改行などの特殊文字を可視化できます。

注意点

catコマンドは非常に便利ですが、大きなファイルには注意が必要です。

例えば数GBあるログファイルに対して使うと、画面に大量の文字が一気に表示されてしまい、ターミナルが見づらくなることがあります。

そのような場合はlessコマンドのようなページ送りできるコマンドを使う方がいいでしょう。

【まどドラ】イベントクエスト・リンクレイドの時短編成

March 25, 2026
Game

日々のイベントクエスト・リンクレイドを時短で終わらせるために私が使っている編成について説明します。

マミ単騎の時短編成

ポートレート・サポートは、後述するマミさんの特長を活かすためにスピード特化にしています。まばゆ未所持の場合は眼鏡ほむらも可能です。

この編成では、 

  • イベントクエスト … すべてのBATTLEをほとんどワンパン

  • リンクレイド … Lv.1ならボスを選ばずワンパン

    が可能です。

なぜマミさんなのか?

マミさんの特長として、

  • 全体攻撃ができること

  • スピードが速く、大抵先行できること

があります。

イベントクエストでは、敵が複数体いることがほとんどなので早く倒したいなら全体攻撃キャラが必須です。また、敵から攻撃を受ける(=敵の攻撃モーションの時間分取られる)ことを避けるためにも、スピードが速いキャラを採用することが重要になります。 マミさん単騎で済むことから、ロード時間の短縮にもなります。

どのイベントクエスト・リンクレイドなら倒せるの?

イベントクエストはイベントの種類に関わらず毎回難易度が一定になっていると思われます。そのため、どのイベントでも安定して倒せるかと思います。

リンクレイドに関しても、どのボスもLv.1なら問題なく倒せます。

敵が非推奨属性だけど倒せる?

イベントクエスト・リンクレイドともに問題なく倒せます。
