Arch Linux をUEFI環境にインストールしたときのメモ

April 23, 2026
Linux

先日、初めてArch Linuxに手を出しました。 Archはインストールが難しそうという偏見からずっと避けてきました。 ですが、最近普段から使っていたdevuanが色々あって使えなくなってしまったので、心機一転、重い腰を上げてArchデビューしてみようと思った次第です。

今回は、Arch linuxインストール中に出会った初見殺しや注意点をメモとして書いていきます。

環境

  • PC
    • Lenovo IdeaPad
  • Arch Linux isoファイル
    • https://mirrors.cat.net/archlinux/iso/2026.04.01/archlinux-x86_64.iso

インストールメディアの作成

この段階でかなり苦戦しまして、、、

最初、iso焼くときはいつもお世話になっているbalena etcherを使って焼いたんですが起動しない、、、というかUEFIに認識されない(Boot orderを変更できない)。

balena etcherがよくないのかと思い、次に試したのがrasberry pi imagerでした。が、それも同じ結果に。。。

何故だろう？考えたときに、一つ心当たりがありました。それは、PCがUEFI環境だったことです。今までBIOSのPCしか使ってこなかったので盲点でした。

で、少し調べると、UEFIのPCでディスクを認識させるためには、パーティションスタイルがGPTでなければいけないということが分かりました。 rasberry pi imagerでisoを書き込んだあとのUSBを確認すると、MBRとなっていました。試しに、windowsでGPT作成したあとでrasberry pi imagerを使ってみましたが、MBRに上書きされてしまっていました。Balena Etcherも同様にMBRに上書きしてしまいます。

なら、rufusを使おう！

rufusを使えば、パーティションスタイルを選べる！！！GPTを選択して、書き込み、UEFIを開くと見事にUSBが認識されていました！

インストールメディア起動

まさかのインストールメディア作成が一番の山場だったんです、、、インストールは結構スムーズにできました！

とりあえずこの公式wikiを読んでそのとおりに進めていけば間違いないと思います。 https://wiki.archlinux.jp/index.php/%E3%82%A4%E3%83%B3%E3%82%B9%E3%83%88%E3%83%BC%E3%83%AB%E3%82%AC%E3%82%A4%E3%83%89

ちなみに、ネットワークはandroidでテザリングしました。有線も無線も面倒なときはテザリングがだいたい解決してくれます。

また、Arch Linuxのインストールで重要なことを色々してくれる(?)このコマンド↓

$ pacstrap -K /mnt base linux linux-firmware

これを実行する前に、UEFIの人は、espを作っておいたほうがいいかも。(AI情報のため真偽は不明だけど)

私は、上記コマンドを実行したあとにespを作成したためなのか、インストール後の3回目の起動くらいで/bootが空っぽになっていました。(謎)

UEFIにも認識されなかったのであせりました。手元にあったArchインストールメディアでespと/boot(grub)を作り直したらなんとか治りました。

テキストエディタ使えなくない?

公式wikiのインストール手順では/etc/locale.confや/etc/hostnameを編集しないといけないときがあるんですが、vimもnanoも使えない (インストールされていない)。

sedやawkも使えない私は最初、どうやって編集すればいいのか分からず迷いましたが、echoの出力をリダイレクトする力技で何とかしました。

例えば、locale-genを実行するために/etc/locale.confのLANG=en_US.UTF-8という行をコメントアウトする作業は、こんな感じです↓

$ echo  'LANG=en_US.UTF-8' > /etc/locale.conf ## >> でもOK

コメントアウトと言っていますが、>を使用しているためファイル内容を上書きしています。

>>を使用すれば元のコメントされている設定を破壊せずにいけますね。

また、私の場合は日本語も使いたいのでLANG=ja_JP.UTF-8も追加してこうしました↓

$ echo -e "LANG=en_US.UTF-8\nLANG=ja_JP.UTF-8" > /etc/locale.conf

お疲れ様

タイトルで「インストールしたときのメモ」とか言いつつ、記事の大半がインストールメディア作成の内容なのは許して。 インストール自体はwikiを見れば特に問題なくできるとは思います。とはいえ、インストール完了後、rebootして最初に起動するかどうかを確認する際の緊張感は半端ないですね。でもちゃんと起動してGrub画面が見れた時はめちゃくちゃ嬉しかったです。

これからX11とか入れて、ネットとかゲームとか色々できる環境にしていくぞー。
