(Linux)ntpdateで時刻を同期する(init)

June 19, 2026
Linux

環境

$ cat /etc/issue
Devuan GNU/Linux excalibur \n \l

linuxで時刻同期です。

私はinit使いなので、よくみんなが使っている(?)systemdで使える同期コマンドは使えません。(timedatectlとか?)

ということで、

ntpdateを使って時刻同期をします

manを少し覗いてみると、色々なオプションがあって使い方が複雑そうですね。

SYNOPSIS
       ntpdate  [-46bBdqsuv]  [-a  key]  [-k keyfile] [-o version] [-t
       timeout] server [...]

ですが、今回は時刻同期をしたいだけなのでオプションは特に必要ないです。

引数にあるserverとして、NTPサーバーを指定すればいいだけです。 NTPサーバーはJSTならntp.nict.jpを使えばいいと思います。

実行例

$ sudo ntpdate ntp.nict.jp

システム時刻設定には管理者権限が必要なことをお忘れなく。

もしうまくいかない場合、時刻が一日以上ずれているなら、date -sで年月日だけでも先に設定しておきましょう。

$ sudo date -s "20260619" # yyyy/mm/dd

日付に大きなズレがあるとNTPの同期がうまくいかないことがあるかもしれないからです。

小言

実験で、date -sを使ってわざと時刻をずらしてntpdate ntp.nict.jpで同期する というのを何度か試していたんですが、なぜか失敗するときがありました。これに関してはなぜななのか検討 がつかない。。。 ちなみに、失敗するときのエラーメッセージはntpdig: no eligible servers。

今ふと思ったんだけど、NTPサーバーに頻繁に接続しすぎて拒否られているだけなのかも…!
